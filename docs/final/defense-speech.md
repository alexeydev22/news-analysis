# Текст доклада к защите

Тема моей курсовой работы: «Разработка автоматической диалоговой системы на основе языковой модели для анализа экономических новостей».

Актуальность темы связана с тем, что поток экономических новостей постоянно растет. Пользователю сложно быстро понять, какие события действительно важны, какой сигнал они несут и как несколько новостей по одной теме связаны между собой. Обычный поиск показывает список документов, но не объясняет их влияние. Обычный чат без источников может дать связный ответ, но его трудно проверить. Поэтому в работе сделан акцент на системе, которая объединяет поиск, машинное обучение, источники и языковую модель.

Цель работы заключалась в разработке прототипа системы, которая принимает вопрос пользователя, находит релевантные экономические новости, оценивает их влияние с помощью моделей машинного обучения и формирует аналитический ответ на основе найденных материалов. То есть система должна быть не просто чат-ботом, а диалоговым аналитическим сервисом, где ответ можно связать с конкретными новостями и результатами моделей.

В качестве данных используется корпус FNSPID. В проекте подготовлена выборка из 50 000 экономических новостей с тремя классами влияния: positive, neutral и negative. На этих данных решается задача многоклассовой классификации: для каждой новости необходимо определить направление экономического сигнала. Такая постановка важна для дальнейшего сценария, потому что найденные новости должны не только выводиться пользователю, но и участвовать в формировании оценки ситуации.

В работе сравниваются три подхода. Первый подход - TF-IDF и Logistic Regression. Это классический интерпретируемый baseline, который хорошо подходит для новостных текстов, потому что в них есть устойчивые лексические признаки: например, слова growth, rose, approved чаще связаны с положительным сигналом, а lower, down, fell - с отрицательным. Второй подход использует эмбеддинги предложений и логистическую регрессию. Он проверяет, насколько полезно смысловое представление текста для этой задачи. Третий подход - легкий transformer-классификатор, который включен как нейросетевая альтернатива.

Лучший результат на текущем корпусе показала модель TF-IDF + Logistic Regression: test accuracy около 0.785 и test macro F1 около 0.717. Для этой задачи macro F1 особенно важен, потому что классы несбалансированы: положительных новостей больше, чем нейтральных и отрицательных. Если смотреть только на accuracy, можно получить завышенное впечатление о качестве модели. Macro F1 показывает, насколько равномерно модель справляется со всеми классами, поэтому эта метрика лучше подходит для анализа влияния новостей.

Полученные метрики нельзя считать финальным промышленным уровнем качества, но они достаточно показательны для курсовой работы. Они демонстрируют, что задача сложнее простого словарного правила: часть новостей имеет неоднозначный контекст, а экономическое влияние зависит от события, сектора и формулировки. При этом TF-IDF baseline оказался рациональным выбором для текущей версии системы, потому что он быстрый, воспроизводимый и объяснимый.

Система работает как ML/RAG-пайплайн. Пользователь задает вопрос в чате. Сервис поиска находит релевантные новости в векторном индексе. Затем сервис анализа применяет ML-модель и определяет сигнал найденных новостей. После этого диалоговый сервис формирует ответ на основе найденного контекста. Важно, что языковая модель получает не только вопрос, но и тексты найденных новостей вместе с результатами анализа. Поэтому итоговый вывод строится на фактах из корпуса, а не на произвольной генерации.

В интерфейсе есть три ключевые страницы. Первая страница - чат. На ней пользователь видит вопрос, ответ, список источников и ход обработки запроса. Это делает работу системы прозрачной: можно увидеть, какие новости были найдены и какие оценки были получены. Вторая страница - ML-отчет. Он показывает качество моделей: accuracy, macro F1, скорость инференса, матрицу ошибок и наиболее характерные признаки для классов. Это важно для защиты, потому что демонстрирует не только интерфейс, но и исследовательскую часть проекта. Третья страница - прогноз. Она группирует новости по похожим темам, агрегирует сигналы моделей и показывает сценарный вывод по группе.

Сценарный прогноз расширяет обычный поиск. Если пользователь смотрит одну новость, он видит только отдельный факт. Если система объединяет несколько новостей по теме, можно понять, какой сигнал преобладает в группе: положительный, нейтральный или отрицательный. Для выбранной группы или новости можно получить текстовый прогноз с помощью подключаемой языковой модели. При этом модель получает полный текст новостей и рассчитанные ML-сигналы, поэтому прогноз опирается на найденные факты.

Архитектура реализована микросервисно, но без избыточного усложнения. Есть сервис новостей, сервис поиска, сервис анализа, сервис диалога, API Gateway и React-интерфейс. Для хранения векторного индекса используется Qdrant, для фоновых задач и событий - Redis, для трекинга экспериментов - MLflow. Backend построен на FastAPI, асинхронном исполнении и слоистой архитектуре. Внутри сервисов разделены домен, сценарии применения, инфраструктура и presentation-слой, а интерфейсы описываются через Protocol. Это позволяет менять модель или способ поиска без полной переработки приложения.

Главный результат работы - готовый прототип диалоговой системы для анализа экономических новостей. Он позволяет загружать и индексировать данные, задавать вопросы в чате, получать ответы с источниками, сравнивать ML-модели и формировать прогноз по темам. Работа соответствует поставленной теме, потому что в центре находится автоматическая диалоговая система, которая использует языковую модель и методы машинного обучения для анализа экономических новостей.

Ограничения текущей версии связаны прежде всего с качеством разметки, дисбалансом классов и ограничениями легкой нейросетевой модели. Экономические новости часто неоднозначны: одна и та же публикация может быть позитивной для одной компании и негативной для другой. Поэтому дальнейшее развитие должно включать очистку разметки, балансировку классов, подбор гиперпараметров, проверку более сильных эмбеддингов и дообучение более мощной transformer-модели. Также можно добавить признаки сектора, компании, даты и источника, чтобы модель лучше учитывала экономический контекст.

В заключение можно сказать, что в работе получилась не только программная реализация, но и воспроизводимый исследовательский контур: есть данные, модели, метрики, отчет, интерфейс и понятные направления улучшения. Система показывает полный путь от корпуса экономических новостей до пользовательского аналитического ответа.
