# Текст доклада к защите

Тема моей курсовой работы: «Разработка автоматической диалоговой системы на основе языковой модели для анализа экономических новостей».

Цель работы заключалась в том, чтобы разработать прототип системы, которая не просто отвечает на вопрос пользователя, а находит релевантные экономические новости, оценивает их влияние с помощью моделей машинного обучения, показывает источники и формирует аналитический прогноз по найденным материалам.

Актуальность темы связана с тем, что поток экономических новостей постоянно растет. Пользователю сложно быстро понять, какие события действительно важны, какой сигнал они несут и как несколько новостей по одной теме связаны между собой. Поэтому в работе была сделана система, совмещающая поиск по корпусу новостей, классификацию тональности и влияния, диалоговый интерфейс и сценарный прогноз.

В качестве данных используется корпус FNSPID. В проекте подготовлена выборка из 50 000 экономических новостей с тремя классами влияния: positive, neutral и negative. На этих данных решается задача многоклассовой классификации: для каждой новости необходимо определить направление экономического сигнала.

В работе сравниваются три подхода. Первый подход - TF-IDF и Logistic Regression. Это классический интерпретируемый baseline, который хорошо работает с новостными текстами, потому что в них много устойчивых лексических признаков: например, слова growth, rose, approved чаще связаны с положительным сигналом, а lower, down, fell - с отрицательным. Второй подход использует эмбеддинги предложений и логистическую регрессию. Он добавляет семантическое представление текста. Третий подход - легкий transformer-классификатор, который был включен как нейросетевая альтернатива.

Лучший результат на текущем корпусе показала модель TF-IDF + Logistic Regression: test accuracy около 0.785 и test macro F1 около 0.717. Для этой задачи macro F1 особенно важен, потому что классы несбалансированы: положительных новостей заметно больше, чем нейтральных и отрицательных. Поэтому нельзя смотреть только на accuracy, нужно учитывать качество по каждому классу.

Система работает как ML/RAG-пайплайн. Пользователь задает вопрос в чате. Сервис поиска находит релевантные новости в векторном индексе. Затем сервис анализа применяет ML-модель и определяет сигнал найденных новостей. После этого диалоговый сервис формирует ответ на основе найденного контекста: в интерфейсе отображается не только итоговый текст, но и источники, оценки релевантности и ход обработки запроса.

Отдельная часть проекта - ML-отчет. Он позволяет сформировать сводку по качеству моделей: сравнить accuracy, macro F1, скорость инференса, посмотреть матрицу ошибок и наиболее характерные признаки для классов. Это важно, потому что система не является «черным ящиком»: результаты можно проверить и объяснить.

Еще один режим - сценарный прогноз. Он группирует новости по похожим темам, агрегирует сигналы моделей и показывает, какое общее влияние преобладает в группе. Для выбранной новости или группы можно получить текстовый прогноз с помощью подключаемой языковой модели через внешний API. При этом прогноз строится не из общего знания модели, а на основе текстов найденных новостей и рассчитанных ML-сигналов.

Архитектура реализована микросервисно, но без избыточного усложнения. Есть сервис новостей, сервис поиска, сервис анализа, сервис диалога, API gateway и React-интерфейс. Для хранения векторного индекса используется Qdrant, для фоновых задач и событий - Redis, для трекинга экспериментов - MLflow. Backend построен на FastAPI, асинхронном исполнении и слоистой архитектуре с разделением домена, сценариев применения и инфраструктуры.

Главный результат работы - готовый прототип диалоговой системы для анализа экономических новостей. Он позволяет загружать и индексировать данные, задавать вопросы в чате, получать ответы с источниками, сравнивать ML-модели и формировать прогноз по темам. Работа соответствует поставленной теме, потому что в центре находится автоматическая диалоговая система, которая использует языковую модель и методы машинного обучения для анализа экономических новостей.

Ограничения текущей версии связаны прежде всего с качеством разметки и несбалансированностью классов. Кроме того, легкая нейросетевая модель требует более тщательной настройки и большего числа экспериментов. В дальнейшем качество можно улучшать через очистку текста, балансировку классов, подбор гиперпараметров, более сильные эмбеддинги, тонкую настройку transformer-модели и добавление экономических признаков, например отрасли, компании, даты и рыночного контекста.

В заключение можно сказать, что в работе получилась не только программная реализация, но и воспроизводимый исследовательский контур: есть данные, модели, метрики, отчет, интерфейс и понятные направления улучшения.
