# Checklist соответствия требованиям

## Тема курсовой работы

Тема: «Разработка автоматической диалоговой системы на основе языковой модели
для анализа экономических новостей».

| Требование темы | Где закрыто |
| --- | --- |
| Автоматическая диалоговая система | `frontend-web`, `api-gateway`, SSE chat flow |
| Языковая модель | LLM-адаптер в `dialog-service`, подключаемая внешняя модель для ответа и прогноза |
| Анализ экономических новостей | `analysis-service`, обученные режимы моделей, датасет FNSPID |
| Работа с источниками | `retrieval-service`, Qdrant, блок источников в UI |
| Воспроизводимая демонстрация | `just demo-up`, `just demo-smoke`, Docker Compose |

## Покрытие исследовательской и инженерной частей

| Направление | Что показано в материалах |
| --- | --- |
| ML/NLP | TF-IDF, embeddings, `tfidf-logreg`, `impact`, `confidence`, `explanation`, retrieval/RAG, Qdrant, LLM-адаптер, MLflow |
| Backend | FastAPI, asyncio, Granian, DDD, Protocol, Dishka, Zapros, Taskiq, Redis, FastStream, SSE, Docker Compose |
| Пользовательский результат | Вопрос в чате, найденные источники, классы влияния, итоговый ответ, timeline обработки |

## Критерии презентации

| Критерий | Баллы | Выполнение |
| --- | ---: | --- |
| Титул | 1 | Слайд 1 содержит тему, вуз, студента, группу, руководителя, город и год. |
| Актуальность | 1 | Слайд 2 раскрывает проблему информационной перегрузки и ценность диалогового анализа. |
| Цели и задачи | 1 | Слайд 3 формулирует цель и задачи проекта. |
| Методы и результаты | 3 | Слайды 4-13 показывают данные, модели, метрики, RAG-пайплайн, архитектуру, UI, ограничения и улучшения. |
| Заключение | 1 | Слайд 14 содержит итоги и направления развития. |
| Список источников | 1 | Слайд 15 содержит основные источники. |
| Оформление | 2 | Использован единый строгий стиль, Times New Roman, сдержанная палитра, таблицы, схема и скриншоты приложения. |

## Пояснительная записка

| Раздел | Выполнение |
| --- | --- |
| Титульный лист | Есть, с данными студента, группы, вуза и руководителя. |
| Введение | Есть актуальность, объект, предмет, цель и задачи. |
| Основная часть | Описаны предметная область, проектирование, реализация и тестирование. |
| Методы и результаты | Описаны RAG-подход, TF-IDF baseline, классификация влияния, LLM-адаптер, микросервисы, DDD, Qdrant, Taskiq, Redis, MLflow, UI, метрики, ошибки и улучшения. |
| Заключение | Есть итоги и направления развития. |
| Источники | Есть список использованных источников с URL, начинается с нового листа. |
